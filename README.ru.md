Постоянная память, растущая идентичность и первопартийная авторизация Anthropic для [opencode](https://opencode.ai). Всё за одну команду.

## Что даёт

- **Трёхслойная память**: Redis для быстрых фактов, ChromaDB для семантического поиска по сессиям, файловая система для структурированных документов
- **Автоинъекция контекста**: каждый промпт обогащается релевантным контекстом из прошлых разговоров
- **Идентичность агента**: убеждения, фокус и наблюдения начинают пустыми и кристаллизуются со временем через работу
- **Затухание памяти**: степенной закон, важное остаётся громким, старый шум стихает, ничего не удаляется
- **Первопартийная авторизация**: OAuth-токены прямо из Claude.app; ваш Max план, а не пул кредитов $200 для сторонних клиентов
- **Оркестрация Sisyphus**: oh-my-openagent с параллельными агентами, загрузкой скиллов и структурированным делегированием

## Требования

- **macOS** (основная) или Linux
- **Python 3.11+**
- **[Claude.app](https://claude.ai/download)**: установлен и залогинен под вашим Max-аккаунтом
- **[Homebrew](https://brew.sh)** на macOS

## Быстрый старт

```sh
# Сначала закройте Claude.app полностью (Cmd+Q)
git clone https://github.com/enkinvsh/crystallized.git
cd crystallized
./install.sh
opencode
```

Установщик сделает всё: Redis, Python-зависимости, opencode CLI, сервер памяти, конфиг и извлечение авторизации.

## Что делает install.sh

- Проверяет зависимости (Git, Python 3.11+, Homebrew на macOS, jq).
- Устанавливает и запускает Redis через Homebrew (или `apt` на Linux), либо использует уже запущенный локальный Redis на порту 6379.
- Устанавливает менеджер пакетов `uv` от astral.sh.
- Устанавливает CLI `opencode` из релизов GitHub, если его ещё нет в PATH.
- Разворачивает скрипты MCP-сервера памяти в `~/.config/opencode/memory/`.
- Устанавливает Python-зависимости в `~/.config/opencode/memory/.venv` через `uv sync --frozen`.
- Создаёт шаблоны идентичности (beliefs, focus, observations, journal) в `~/.config/opencode/memory/notes/`.
- Записывает или сливает `~/.config/opencode/opencode.json` с записью MCP памяти и pre-prompt хуками. Перед слиянием установщик делает резервную копию существующего файла.
- На macOS извлекает OAuth-токены Claude.app через Keychain и записывает их в `~/.local/share/opencode/auth.json`.

## Что install.sh НЕ делает

- Не изменяет Claude.app и его файлы.
- Не трогает ваши rc-файлы оболочки. Если после установки `opencode` не оказался в PATH, установщик печатает однострочную подсказку `export PATH=...`, чтобы вы добавили её сами.
- Не отправляет данные. Никакой телеметрии, аналитики или удалённого логирования.
- Не работает на Windows. WSL не тестировался.
- Не находит Claude.app вне `/Applications/Claude.app`. Если Claude установлен в нестандартное место, запустите `python3 auth/extract_token.py` вручную.

## Оговорки

- При извлечении авторизации Keychain может запросить пароль вашей учётки macOS. Выберите "Always Allow", чтобы избежать повторных запросов.
- На Linux шаг автоматической авторизации пропускается. Нужно извлечь токены с мака, использовать API-ключ напрямую или принять маршрутизацию через сторонний пул кредитов.
- Установщик рассчитан на однопользовательский Mac. Многопользовательские общие установки не поддерживаются.
- Холодный старт ChromaDB может занять от 10 до 30 секунд при первом MCP-вызове, пока скачивается модель sentence-transformer.

## Необязательные переменные окружения времени выполнения

Все переменные окружения времени выполнения необязательны. Если их не задавать, значения по умолчанию сохранят поведение v1.0.

| Переменная | Назначение |
|---|---|
| `REDIS_URL` | Полный URL подключения к Redis. |
| `REDIS_HOST` | Хост Redis, если `REDIS_URL` не задан. |
| `REDIS_PORT` | Порт Redis, если `REDIS_URL` не задан. |
| `REDIS_DB` | Номер базы данных Redis. |
| `OPENCODE_MEMORY_SOCKET` | Путь к MCP-сокету для хуков памяти. |
| `OPENCODE_MEMORY_NOTES_DIR` | Каталог заметок для сохранённых документов и файлов идентичности. |
| `OPENCODE_MEMORY_CHROMA_DIR` | Каталог постоянного хранения ChromaDB. |

## Обновление с v1.0

Заберите новый код, затем снова запустите `./install.sh`. Установщик сделает резервную копию `opencode.json` перед слиянием новых memory hooks и MCP-конфига.

Новые переменные окружения необязательны. Если их не задавать, значения по умолчанию сохранят поведение v1.0.

Если что-то сломалось, запустите `./uninstall.sh`, затем снова запустите `./install.sh`.

Полные заметки о релизе смотрите в [CHANGELOG.md](CHANGELOG.md).

## Зачем это

Anthropic определяет сторонние клиенты и направляет их API-запросы в отдельный пул кредитов ($200) вместо вашей Max-подписки. Коммьюнити-плагины авторизации получают OAuth-токены со сторонним `client_id`, поэтому каждый запрос помечается.

Crystallized извлекает токены напрямую из Claude.app. Они несут собственный `client_id` Claude, поэтому Anthropic считает ваши сессии opencode первопартийными. Лимиты Max-плана работают штатно.

## Как работает

### Память

`memory-inject.py` запускается как хук перед каждым промптом. Ищет релевантный контекст по трём слоям и вставляет его:

| Слой | Движок | Назначение |
|---|---|---|
| Факты | Redis | Имена, решения, предпочтения, мгновенный key/value |
| Семантика | ChromaDB | Векторный поиск по всему, что агент когда-либо запоминал |
| Документы | Файловая система | Архитектурные заметки, чеклисты, саммари сессий |

Затухание работает по степенному закону. Воспоминания никогда не удаляются, они становятся тише.

### Идентичность

`own-voice.py` инъектирует эволюционирующую идентичность агента в каждый промпт. Убеждения, фокус и наблюдения начинают как пустые файлы и заполняются по мере работы. Личность заработана опытом, а не задана конфигом.

### Авторизация

`auth/extract_token.py` расшифровывает safeStorage Claude.app (Electron, AES-128-CBC через macOS Keychain), извлекает OAuth-токены и записывает их в `auth.json` opencode. Обновление токена opencode делает сам, никакой auth-плагин при работе не нужен.

## Архитектура

```
~/.config/opencode/
├── opencode.json              # MCP-серверы, плагины
└── memory/
    ├── server.py              # MCP сервер памяти (Redis + ChromaDB + fs)
    ├── memory-inject.py       # Хук: инъекция контекста
    ├── own-voice.py           # Хук: инъекция идентичности
    ├── pyproject.toml         # Python-зависимости
    ├── chroma_db/             # Векторная БД (генерируется)
    ├── notes/self/            # Идентичность агента (генерируется)
    │   ├── beliefs.md
    │   ├── focus.md
    │   └── observations.md
    └── identity.json          # Карта громкости (генерируется)

~/.local/share/opencode/
└── auth.json                  # OAuth-токены (из Claude.app)
```

## Проблемы и решения

**Ошибка "Third-party apps"**, неправильный токен. Закройте Claude.app, затем:
```sh
python3 auth/extract_token.py
```
Если несколько workspace, попробуйте каждый индекс.

**Memory MCP красный**, `redis-cli ping` должен вернуть PONG. Также проверьте что путь к `uv` в `opencode.json` абсолютный. Установщик прописывает автоматически, но ручные правки могут сломать.

**Keychain access denied**, нужен терминал с GUI, не чистый SSH. Или разблокируйте сначала:
```sh
security unlock-keychain ~/Library/Keychains/login.keychain-db
```

**Linux**, извлечение авторизации только для macOS (Claude.app). Перенесите токены с мака, используйте API-ключ напрямую или примите маршрутизацию через сторонний пул.
